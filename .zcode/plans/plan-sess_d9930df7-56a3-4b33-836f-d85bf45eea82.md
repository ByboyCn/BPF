# bpf 跨平台 UI 库 —— 总体设计与 M1 实施计划

## 核心设计原则(已确认)

- **语言**: C# + NativeAOT
- **TFM 策略**: 核心 + Windows 实现层用 `netstandard2.1`;**仅 sample 应用**用 `net10.0` 单框架并开启 AOT
- **控件**: 完全自研,不引用 Avalonia 程序集,API 风格向 Avalonia 靠拢
- **平台**: 跨平台抽象层 + 仅 Windows 实现
- **渲染**: Direct2D + DirectWrite + D3D11(系统组件,零外部依赖)
- **界面**: 代码优先(C# 构建控件树)
- **AOT 约束**: 禁用 Reflection.Emit、最小化反射、用泛型静态注册替代运行时代码生成

### 关于 netstandard2.1 + NativeAOT

- netstandard2.1 库**本身不能单独被 AOT 编译**,但被 net10.0 AOT 应用引用消费时**完全支持**:AOT 分析器会穿透分析其 IL。
- 因此代码必须全程保持 AOT 友好(无 `Emit`、最小反射、泛型实例化优先)。这是硬性编码纪律,贯穿所有库。
- netstandard2.1 拥有 `Span<T>`/`Memory<T>`/`HashCode` 等现代 API,够用;若用到 C# `record`/`init` 的运行时支持,手写等价代码或 polyfill,不依赖新运行时。

> 关于"只留一个 dll":NativeAOT 编译 Win 应用产出是单个原生 **`.exe`**(所有 .NET 依赖内联编译),不产出托管 dll,不带额外 dll。D2D1/DWrite/D3D11 都是 Windows 系统组件,不随程序分发。M1 先按单 exe;如需 native dll 形态后续加 `/p:PublishAotAsLibrary=true`。

## 关键技术设计

### A. AOT 友好的属性系统

用**泛型实例化**保证 NativeAOT 保留类型元数据,不用 Avalonia 的运行时反射注册:

```csharp
public class Button : Control {
    public static readonly StyledProperty<string> ContentProperty
        = StyledProperty<string>.Register<Button>(nameof(Content), "");
    public string Content {
        get => GetValue(ContentProperty);
        set => SetValue(ContentProperty, value);
    }
}
```

`StyledProperty<T>.Register<TOwner>` 返回每个声明独立的静态实例;`GetValue/SetValue` 用泛型字典(`Dictionary<BpfProperty, object>`)而非反射。NativeAOT 对每个 `StyledProperty<T>` 的不同 owner 都会生成独立代码并保留。

### B. 平台抽象(跨平台命脉)

`Bpf` 核心层只定义接口(`IPlatformRenderInterface`/`IPlatformWindow`/`IDrawingContext`),控件/布局/样式全部针对接口编程。`Bpf.Windows` 提供实现,通过 `Application.UseWindows()` 注册。后续 Linux/macOS 实现同名接口,核心代码零改动。

### C. 渲染管线(D2D1 on DXGI)

1. D3D11 创建 device
2. DXGI swapchain(FLIP_DISCARD)绑到 HWND
3. D2D1 factory 在 DXGI backbuffer 上创建 `ID2D1DeviceContext`
4. `IDrawingContextImpl` 包装 D2D context,提供 `DrawRectangle/DrawText/FillGeometry`
5. 控件 `Render(DrawingContext)` 调用这些方法,完全不感知 D2D

### D. 消息循环与 AOT

Win32 经典循环 `GetMessage → TranslateMessage → DispatchMessage`,WndProc 回调驱动输入和重绘。不依赖运行时动态特性,避免 AOT 反射风险。

### E. COM Interop 在 netstandard2.1 下的处理

D2D1/D3D11/DWrite 都是经典 COM。在 netstandard2.1 下用手写 `[ComImport]` 接口 + `[DllImport]` 工厂函数获取首个指针,再用 `Marshal.QueryInterface`/VTable 调用。避免依赖 `CsWin32`/`WinRT` 这些需要新 TFM 的工具链,**保证 netstandard2.1 可编译**。

## M1 里程碑(Minimal Viable —— 端到端跑通)

**目标**: `dotnet publish -r win-x64 -c Release /p:PublishAot=true` 产出单个 exe,运行后弹出窗口,内含可点击 Button + 显示文字 TextBlock,点击 Button 改变 TextBlock 内容。

### M1 交付清单(按依赖顺序)

1. **`Directory.Build.props`** —— 统一 `LangVersion=latest` / `Nullable=enable`(不设全局 TFM)
2. **`src/Bpf/Bpf.csproj`** —— `netstandard2.1` 纯核心库
3. **`src/Bpf` 核心层**:
   - 几何类型(Size/Rect/Vector/Thickness/Color)
   - 属性系统(BpfProperty / StyledProperty<T> / PropertyValueStore)
   - Controls 基类链:Visual → Control → Layoutable
   - Platform 接口族
   - Media:SolidColorBrush
   - Layout:Measure/Arrange 协议、简单 LayoutManager
   - Application 入口
4. **`src/Bpf.Windows/Bpf.Windows.csproj`** —— `netstandard2.1`(纯手写 P/Invoke + COM)
5. **`src/Bpf.Windows` 实现层**:
   - Interop: user32 / d3d11 / dxgi / d2d1 / dwrite
   - D3D11 + DXGI swapchain 装配
   - D2D1RenderTarget + D2D1DrawingContext
   - DWriteFactory + 文本格式缓存
   - WindowImpl: RegisterClass/CreateWindow/WndProc
   - Win32Dispatcher: 主循环
6. **Button / TextBlock** —— 标准控件,完整 Measure/Arrange/Render
7. **`samples/Bpf.Samples.HelloWorld`** —— `net10.0` + `<PublishAot>true</PublishAot>`,一个按钮 + 文本块
