# bpf M2 实施计划:键盘 / 焦点 / 路由事件

## 目标
为 bpf 引入**正式的路由事件系统**(Bubble + Tunnel)、**键盘输入**、**焦点管理**。
完成后,控件支持键盘事件冒泡、焦点切换(Tab/方向键),为 M3 的 TextBox/CheckBox 等交互控件打好地基。

## 核心设计决策

### A. 路由事件(AOT 友好,仿 Avalonia)
用 `RoutedEvent.Register<TOwner>(...)` 泛型静态注册,每个声明是独立静态字段(NativeAOT 对每个泛型实例生成独立代码),**不使用运行时反射 registry**。API 风格:
```csharp
public static readonly RoutedEvent<KeyEventArgs> KeyDownEvent =
    RoutedEvent.Register<Button>(nameof(KeyDown), RoutingStrategies.Bubble);
```
路由策略:`Bubble`(自下而上,默认)、`Tunnel`(自上而下,前缀 Preview)、`Direct`(不路由)。

### B. 事件路由实现
`Control` 基类新增:
- `AddHandler<TArgs>(RoutedEvent<TArgs>, EventHandler<TArgs>)` / `RemoveHandler`
- 路由走 `BuildEventRoute(target)` 构造从 root → target 的控件链
- Bubble:从 target 往上逐个调 `OnXxx(e)` + 订阅的 handler;`e.Handled = true` 中止
- Tunnel:反向(目前 M2 主要实现 Bubble,Tunnel 留接口)

```csharp
case WM_KEYDOWN: case WM_SYSKEYDOWN:
    KeyDown?.Invoke(this, new KeyEventArgs(Win32KeyMap.MapKey(wParam), GetModifiers()));
    return IntPtr.Zero;  // 不转给 DefWindowProc,避免系统 beep
case WM_KEYUP: case WM_SYSKEYUP:
    KeyUp?.Invoke(this, ...);
    return IntPtr.Zero;
case WM_CHAR:

    return IntPtr.Zero;
```

## 关键技术点

### 1. 路由事件如何与 AOT 兼容
`RoutedEvent<TArgs>` 每个声明是独立静态字段。`AddHandler<TArgs>(RoutedEvent<TArgs>, handler)` 用 `Dictionary<RoutedEvent, List<Delegate>>` 存储订阅。**零反射** —— NativeAOT 对每个 `RoutedEvent<TArgs>` 泛型实例生成独立代码。与 `StyledProperty<TValue>` 设计同构。

### 2. 递归命中测试
`Control.HitTest` 已支持递归(StackPanel 转发给 children)。新增 `Window.FindHitTestTarget(Point)` 返回**最深命中的叶子控件**,然后对它 `RaiseEvent`,Bubble 自动冒泡到所有祖先。这修掉 M1 的 flat 一层 bug。

### 3. 焦点管理(M2 简化版)
- `FocusManager.Focused`(静态,全局唯一焦点控件)
- `Control.Focus()` → 设 `IsFocused=true`,前一个焦点控件 `IsFocused=false`,各自触发 OnGotFocus/OnLostFocus
- Tab 键在 `Window` 层处理:遍历 `IsFocusable` 子控件,循环切换
- M2 不做 TabIndex/方向键导航(M3 再加)

### 4. 路由与 Bounds 坐标
冒泡时 `e.Source` 始终是最初命中的控件;每个控件收到的 PointerEventArgs.Position 是**相对于自己**的坐标。派发前由 Window 做坐标转换。

## 风险
- 路由事件改造会动 Control 基类,影响所有现有控件。我会保持 M1 的 `OnPointerPressed` 虚方法 + CLR `event` 的便捷订阅方式不变,路由是**额外的**派发路径,不破坏现有 Button.Click 的用法。
- Tab 焦点遍历需要维护"可聚焦控件列表",涉及控件树遍历。M2 用简单深度优先,M3 优化。
