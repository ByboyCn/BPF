# bpf M4 实施计划:数据绑定 + ListBox

## 目标
为 bpf 引入 **数据绑定系统**(INotifyPropertyChanged + 属性路径绑定)和 **ListBox** 控件(列表选择,ItemsSource 绑定数据集合)。

## 分两阶段

### 阶段 A:数据绑定系统

#### 核心类型

1. **INotifyPropertyChanged**(`src/Bpf/Data/INotifyPropertyChanged.cs`)
   - 数据对象实现:`event PropertyChangedEventHandler? PropertyChanged`
   - AOT 完全友好(纯事件,无反射)

2. **ObservableCollection<T>**(`src/Bpf/Data/ObservableCollection.cs`)
   - 继承 `List<T>`,实现 INotifyCollectionChanged
   - Add/Remove 时触发 CollectionChanged 事件
   - ListBox 的 ItemsSource 用它

3. **Binding**(`src/Bpf/Data/Binding.cs`)
   - 描述一次绑定:Source(数据对象)、Path(属性路径字符串)、Mode(OneWay/TwoWay/OneTime)
   - 核心方法:`object? Evaluate()` —— 从 Source 沿 Path 读取当前值

4. **BindingExpression**(`src/Bpf/Data/BindingExpression.cs`)
   - 绑定的运行时实例,挂在 StyledProperty 上
   - 监听 Source 的 PropertyChanged,变化时刷新目标控件的属性值
   - 管理绑定生命周期(attach/detach)

#### 属性路径解析(AOT 友好)
`"User.Age"` → 用反射逐段读 `source.GetType().GetProperty("User")?.GetValue(source)`。
NativeAOT **支持反射读取**(PropertyInfo.GetValue),但需要:
- 数据对象类型不被 trim(用户数据类通常是 public class,默认保留)
- 属性是 public(被 trimmer 默认保留)
- 路径中每个属性的 getter 可访问

如果 trim 把属性 getter 裁剪了,绑定会返回 null(优雅降级,不崩溃)。用户可通过 `[DynamicallyAccessedMembers(DynamicallyAccessedMemberTypes.PublicProperties)]` 标注数据类型保证保留。

#### Control 接入
- `Control.SetBinding(StyledProperty, Binding)` —— 在控件属性上挂绑定
- `GetValue` 改造:有绑定时先从绑定读值(优先于样式,仅次于本地值)
- 查找顺序:**本地值 → 绑定 → 样式 → 默认值**

#### 新增文件
- `src/Bpf/Data/INotifyPropertyChanged.cs`
- `src/Bpf/Data/NotifyPropertyChangedExtensions.cs`(SetProperty 助手)
- `src/Bpf/Data/ObservableCollection.cs` + `INotifyCollectionChanged` + `NotifyCollectionChangedEventArgs`
- `src/Bpf/Data/Binding.cs`
- `src/Bpf/Data/BindingExpression.cs`
- `src/Bpf/Data/BindingMode.cs`

#### 修改
- `src/Bpf/Controls/Control.cs` — 加 `_bindings` 字典 + `SetBinding` 方法 + GetValue 改造

### 阶段 B:ListBox 控件

#### 设计
- **属性**:`ItemsSource`(IEnumerable)、`SelectedItem`(object?)、`SelectedIndex`(int)
- **ItemsSource 绑定**:支持 ObservableCollection<T>,集合变化时重建列表项
- **列表项**:每个数据项生成一个 TextBlock(或用户自定义 ItemTemplate,M4 先用默认文本)
- **选择**:点击列表项设 SelectedItem,触发 SelectionChanged 事件
- **滚动**:M4 先做固定高度(不滚动),ScrollViewer 留 M4.1

#### 渲染
- 每个 item 是一行:选中项高亮背景(蓝色),其余白底
- 文字用数据项的 ToString()(默认)

#### 新增文件
- `src/Bpf/Controls/ListBox.cs`
- `src/Bpf/Controls/ListBoxItem.cs`(内部:单个列表项控件)

### 阶段 C:sample 演示

重写 sample:
- 定义一个数据模型 `Person { Name, Age }` 实现 INotifyPropertyChanged
- 用 ObservableCollection<Person> 作为 ListBox 的 ItemsSource
- TextBox 双向绑定到选中 Person 的 Name
- Slider 绑定到选中 Person 的 Age
- 点击 ListBox 项切换选中,TextBox/Slider 同步更新
- 演示 OneWay 绑定 + 双向绑定

## 关键技术点

### 1. 绑定的 AOT 友好性
- INotifyPropertyChanged 是纯接口事件,AOT 完美支持
- 属性路径用反射 **读取**(PropertyInfo.GetValue),NativeAOT 支持
- 绑定更新靠事件驱动(PropertyChanged),不靠轮询
- TwoWay 绑定的反向写入(source.Property = value)也用反射 SetValue,AOT 支持

### 2. GetValue 改造的影响
增加绑定层后查找顺序变为:本地值 → 绑定 → 样式 → 默认值。需确保不破坏现有控件(Button 的 Background 等仍是本地默认值,不受影响,因为绑定为 null 时不参与)。

### 3. ListBox 的虚拟化
M4 不做虚拟化(所有项都创建控件)。数据量大时性能差,但 M4 优先功能正确性。ScrollViewer + 虚拟化留 M4.1。

### 4. 反向绑定(TwoWay)的触发
控件属性变化时(如 TextBox.Text 被 OnTextInput 修改),需要反向写回 Source。这通过监听控件自身的属性变化(OnPropertyChanged)触发。

## M4 验收清单
- [ ] `dotnet build` 全绿,AOT 警告 ≤ 可接受(反射绑定的 trim 警告可标注处理)
- [ ] INotifyPropertyChanged:数据对象属性变化时,绑定的 UI 自动刷新
- [ ] 属性路径绑定:`Bind(textBox, TextBox.TextProperty, person, "Name")` 生效
- [ ] TwoWay 绑定:UI 改变 → 数据更新(TextBox 输入改 Person.Name)
- [ ] ListBox:ItemsSource 显示列表,点击选中,SelectedItem 绑定
- [ ] ObservableCollection:Add/Remove 时 ListBox 自动更新
- [ ] sample:ListBox + TextBox/Slider 双向绑定 Person
- [ ] `dotnet publish -r win-x64 -c Release /p:PublishAot=true` 单 exe,稳定 5/5

## 不在 M4 范围
- ScrollViewer/滚动(M4.1)
- ComboBox(下拉选择,M4.1)
- Menu/ToolBar(后续)
- 绑定的 Converter(值转换,M4.1)
- 绑定的 UpdateSourceTrigger(Explicit,M4.1)
- 虚拟化(大数据量,后续)
- 相对源绑定(ElementName/FindAncestor,M5)

## 风险
- 反射绑定的 trim 兼容性:需要测试 AOT 下 PropertyInfo.GetValue 是否正常工作。如果被 trim,需要 DynamicallyAccessedMembers 标注或源生成器。
- TwoWay 绑定的循环触发:控件改 → 数据改 → 通知 → 控件改。需要防重入标志。
- ListBox 无滚动:如果 items 多于可见区,只能看到前面的。M4 先验证选择逻辑正确。
