# bpf M5 修订计划:用 SkiaSharp 替换 D2D1/DWrite/WIC 渲染后端

## 核心变更
**删除所有 D2D1/DWrite/WIC COM 互操作**,改用 **SkiaSharp** 作为渲染后端。
SkiaSharp 是跨平台图形引擎(Windows/Linux/macOS 各有 native lib),支持:
- 矩形/圆角矩形/路径绘制(SKCanvas.DrawRect/DrawRoundRect/DrawPath)
- 文字渲染 + 度量(SKFont + SKPaint.MeasureText)
- 图片解码(SKBitmap.Decode 支持 PNG/JPEG/BMP/GIF)
- 抗锯齿
- 不依赖 COM,纯 P/Invoke 到 native lib(libSkiaSharp.dll / libSkiaSharp.so / libSkiaSharp.dylib)

## 架构变化

### 之前(M1-M4.1)
```
控件 → IDrawingContext → Win32DrawingContext → D2D1 COM vtable → d2d1.dll
                                         → DWrite COM → dwrite.dll
                                         → WIC COM → windowscodecs.dll
```

### 之后(M5)
```
控件 → IDrawingContext → SkiaDrawingContext → SKCanvas → libSkiaSharp.dll
                   → SkiaTextFormat → SKFont/MeasureText
                   → SkiaBitmap → SKBitmap.Decode
```

**核心库(src/Bpf/)零改动** —— 所有接口(IPlatformRenderInterface/IDrawingContext/IRenderTarget/IPlatformBrush/IPlatformTextFormat/IPlatformBitmap)签名不变。

## SkiaSharp 的窗口呈现

SkiaSharp 需要把 SKSurface 的内容呈现到 HWND。两种方式:
1. **SKGLSurface**(GPU,需要 GL 上下文)— 性能好但需要创建 OpenGL 上下文
2. **SKSurface.Create(bitmap)**(CPU 光栅化到 SKBitmap,然后 BitBlt 到 HDC)— 简单,不需要 GL 上下文

M5 先用方式 2(CPU 光栅化 + BitBlt),简单可靠。后续可以升级到 GL。

Present 流程:
```
SkiaRenderTarget.Present():
  1. SKSurface 上的绘制已完成
  2. 获取 SKBitmap 的像素数据
  3. 用 gdi32 SetDIBitsToDevice 把像素 blit 到窗口 HDC
```

## 优势
1. **跨平台统一**:将来 Linux/macOS 后端只需替换 native lib,核心渲染代码完全复用
2. **不依赖 COM**:消除了所有 COM vtable 坑(slot 错位、IID 抄错、AOT marshalling 问题)
3. **图片原生支持**:SKBitmap.Decode 直接支持 PNG/JPEG/BMP/GIF,不需要 WIC
4. **文字原生支持**:SKFont/MeasureText 替代 DWrite,支持任意字体

## 风险
- SkiaSharp 的 native lib(libSkiaSharp.dll ~6MB)需要随程序分发,打破了"只留一个 exe"的目标。但可以通过 PublishSingleFile 合并。
- AOT 兼容性需验证(SkiaSharp 大量使用 P/Invoke,应该 AOT 友好)。
