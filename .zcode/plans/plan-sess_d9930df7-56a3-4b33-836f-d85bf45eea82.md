# bpf M4.1 实施计划:ScrollViewer + ComboBox + Converter + Label/Image

## 目标
为 bpf 引入 **ScrollViewer**(滚动)、**ComboBox**(下拉选择)、**IValueConverter**(绑定转换)、**Label**(标签)、**Image 占位**(控件框架 + 纯色渲染,真图片加载留后续)。

## 分五部分

### 第 1 部分:鼠标滚轮事件(ScrollViewer 的前提)

全链路添加滚轮支持(当前完全缺失):
1. **User32.cs** — 加 `WM_MOUSEWHEEL = 0x020A` 常量 + `ScreenToClient` P/Invoke(滚轮坐标是屏幕坐标)
2. **MouseWheelEventArgs**(`src/Bpf/Platform/MouseWheelEventArgs.cs`)— 继承 RoutedEventArgs,含 `Point Position` + `double Delta`
3. **IPlatformWindow.cs** — 加 `event EventHandler<MouseWheelEventArgs>? MouseWheel`
4. **WindowImpl.cs** — WndProc 加 `case WM_MOUSEWHEEL`,解析 wParam 高字(有符号 delta),ScreenToClient 转客户区坐标,触发 MouseWheel
5. **Control.cs** — 加 `MouseWheelEvent` 路由事件 + `OnMouseWheel` 虚方法 + CLR event
6. **Window.cs** — 订阅 MouseWheel,派发给命中控件(走 Bubble 路由)

### 第 2 部分:ScrollViewer

- **单 Child 装饰容器**(参照 Border 的 Child/AttachNonPanelChildren 模式)
- **属性**:VerticalOffset(double)、HorizontalOffset(double,先不用)、VerticalScrollBarVisibility
- **Measure**:子控件用 Infinity 高度约束(让其报告完整高度)
- **Arrange**:子控件排在 (0, -VerticalOffset) 偏移处
- **Render**:PushClip(viewport) → PushTranslate(0, -offset) → 子.Render → PopTransform → PopClip
- **滚动**:OnMouseWheel 改 VerticalOffset(delta 正值向上滚 → offset 减小)
- **HitTest**:只在 viewport 范围内命中,偏移坐标转到子控件空间

### 第 3 部分:ComboBox

- **属性**:ItemsSource(IEnumerable)、SelectedItem、SelectedIndex、Text(显示当前选中)
- **结构**:Button 风格的触发器(点击展开)+ 弹出层(暂用 Border + ListBox)
- **展开/收起**:点击切换 `_isDropdownOpen`,展开时渲染 ListBox 列表;选择后收起
- **M4.1 简化**:弹出层直接在 ComboBox 下方渲染(不用独立 popup 窗口),用 PushClip 限制可视区
- **选择**:委托给内部 ListBox 的选择逻辑

### 第 4 部分:IValueConverter

- **IValueConverter**(`src/Bpf/Data/IValueConverter.cs`):
  ```
  object? Convert(object? value, Type targetType, object? parameter)
  object? ConvertBack(object? value, Type targetType, object? parameter)
  ```
- **Binding** 加 `Converter` + `ConverterParameter` 属性
- **BindingExpression.Refresh**:Evaluate 后、SetValueInternal 前,应用 Converter.Convert
- **BindingExpression.PushBack**:SetValue 前,应用 Converter.ConvertBack
- **内置转换器**:`BoolToVisibilityConverter`(但 bpf 还没有 Visibility 枚举,M4.1 先做接口 + 一个示例转换器)

### 第 5 部分:Label + Image 占位

- **Label**:简单文本控件,关联另一个控件的 Target(点击 Label 聚焦 Target)。M4.1 先做纯文本显示(等同 TextBlock,加 Target 属性)
- **Image 占位**:定义 `Image` 控件类 + `Source` 属性(object?),M4.1 渲染用纯色矩形占位(标注 "图片渲染留后续")。真正 WIC + D2D1 DrawBitmap 留 M5(避免大量 COM 互操作风险)

## 关键技术点

### 1. 滚轮坐标转换
WM_MOUSEWHEEL 的 lParam 是**屏幕坐标**(不是客户区),需要 `ScreenToClient(hwnd, &point)` 转换。delta 在 wParam 高字(short,有符号),标准滚轮一格 = 120。

### 2. ScrollViewer 的 Clip+Translate 顺序
```
PushClip(viewport 0,0,W,H)  // 裁剪到可视区
PushTranslate(0, -VerticalOffset)  // 内容向上偏移
child.Render()
PopTransform()
PopClip()
```
子控件在偏移后的坐标渲染,超出 viewport 的部分被裁剪。D2D1 的 clip 在当前变换空间解释,这个顺序是正确的。

### 3. ScrollViewer 命中测试
需要在 viewport 范围内做命中测试,且把命中坐标加上 VerticalOffset 转换到子控件空间。重写 HitTest:先检查 point 是否在 viewport 内,不在则返回 false;在则把 point.Y += VerticalOffset 后转发给 child.HitTest。

### 4. ComboBox 的弹出层
M4.1 不做独立 popup 窗口(需要第二个 HWND,复杂)。弹出层直接在 ComboBox 的 Render 里画:展开时在 ComboBox 下方渲染一个 ListBox 区域。用 PushClip 防止超出窗口。

## 不在 M4.1 范围
- 真正的图片加载/渲染(WIC + D2D1 DrawBitmap,留 M5)
- ScrollBar 控件(可视滚动条,留后续,滚轮先够用)
- ComboBox 的独立 popup 窗口(留后续)
- Visibility 枚举(留后续,配合 BoolToVisibilityConverter)
- 水平滚动(先做垂直)

## 风险
- 鼠标滚轮的 ScreenToClient 坐标转换是新增 P/Invoke,需测试 AOT 兼容
- ComboBox 弹出层的非独立窗口方案在边界情况下可能被裁剪,但 M4.1 可接受
- Image 占位只是框架,用户如果期望真图片会失望,需明确标注
