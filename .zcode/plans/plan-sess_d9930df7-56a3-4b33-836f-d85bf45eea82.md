# bpf M3 实施计划:样式系统 + 交互控件

## 目标
为 bpf 引入 **样式系统**(Style/Setter,代码优先)和 **4 个交互控件**(TextBox/CheckBox/RadioButton/Slider)。

## 分两阶段

### 阶段 A:样式系统

#### 设计
- **Style**:针对某控件类型的一组属性覆盖。`Style<T>` where T:Control,含 `Setters` 列表。
- **Setter**:一个 `(StyledProperty, value)` 对。泛型 `Setter<TValue>(StyledProperty<TValue>, TValue)` 保证类型安全。
- **应用机制**:Style 挂在 Control 或 Window 上(`Styles` 列表)。`Control.GetValue` 查找顺序:**本地值 → 匹配的 Style Setter → 默认值**。
- **匹配规则**:Style<T> 匹配类型 T 及其子类(WPF 风格)。最具体匹配优先。

#### GetValue 改造(关键)
当前:`本地值 → 默认值`。改为:`本地值 → Style → 默认值`。
```
GetValue(property):
  1. 本地有值 → 返回本地值
  2. 遍历自身 + 祖先的 Styles,找匹配的 Setter → 返回 Setter 值
  3. 返回 property.DefaultValue
```
为保证 AOT 友好,Style 查找用类型比较(不用反射),Setter 存为 `(StyledProperty, object)`。

#### 新增文件
- `src/Bpf/Styling/Style.cs` — `Style<T> where T:Control`,`IReadOnlyList<Setter> Setters`,`Add(Setter)`
- `src/Bpf/Styling/Setter.cs` — `Setter` 基类 + `Setter<TValue> : Setter`

#### 修改
- `src/Bpf/Controls/Control.cs` — 加 `List<Style> Styles` 属性;改造 `GetValue` 加 Style 查找
- `src/Bpf/Controls/Window.cs` — Window 的 Styles 默认应用到所有后代

### 阶段 B:4 个交互控件

#### 1. TextBox(最复杂)
- **属性**:Text(string)、CaretIndex(int)、Foreground/Background/FontFamily/FontSize、IsReadOnly
- **输入**:`OnTextInput` 追加字符;`OnKeyDown` 处理 Backspace/Delete/Left/Right/Home/End
- **光标**:闪烁的竖线(FillRectangle + 定时器)。M3 用简单的"聚焦时显示光标"
- **渲染**:背景 + 边框 + 文字 + 光标竖线;文字超出时用 PushClip 裁剪
- **焦点**:点击获得焦点,IsFocusable=true

#### 2. CheckBox
- **属性**:IsChecked(bool)、Content(string)
- **交互**:点击切换 IsChecked,触发 Checked/Unchecked 事件
- **渲染**:方框(勾选时画 ✓ 或填充)+ 文字
- **路由事件**:CheckedEvent / UncheckedEvent(或统一的 IsCheckedChangedEvent)

#### 3. RadioButton
- **属性**:IsChecked(bool)、GroupName(string)
- **交互**:点击设 IsChecked=true,同 GroupName 的其他 RadioButton 自动取消
- **渲染**:圆圈(选中时画中心点)+ 文字
- **互斥逻辑**:点击时遍历控件树找同 GroupName 的 RadioButton,设其它为 false

#### 4. Slider
- **属性**:Minimum/Maximum/Value(double)、Orientation
- **交互**:点击/拖拽轨道设置 Value
- **渲染**:轨道(FillRectangle 细长矩形)+ 滑块(FillRoundedRectangle)
- **路由事件**:ValueChangedEvent

### 阶段 C:sample 演示

重写 sample 展示所有新控件 + 样式:
- 全局样式(给所有 Button 设统一配色)
- TextBox(可输入文字)
- CheckBox + RadioButton(演示互斥)
- Slider(拖拽调值,绑定到某 TextBlock 显示)

## 关键技术点

### 1. 样式查找的 AOT 友好性
`Style<T>` 的 T 在编译期确定。Control.GetValue 查找时,用 `control is T` 模式匹配(C# is 操作符,AOT 友好)。Setter 存储 `(StyledProperty, object)`,读取时强转(同 StyledProperty<TValue> 的用法)。

### 2. TextBox 的文字输入链路
Window.OnPlatformTextInput → FocusManager.Focused.RaiseEvent(TextInputEvent) → TextBox.OnTextInput。
TextBox.OnTextInput:`Text = Text.Insert(CaretIndex, e.Text); CaretIndex += e.Text.Length`。
OnKeyDown 处理编辑键:Backspace 删前一个,Delete 删后一个,Left/Right 移动光标。

### 3. CheckBox 的 ✓ 渲染
没有 DrawLine,用两条 FillRectangle(细矩形旋转模拟)不可行(只有 PushTranslate)。
M3 方案:勾选时用 FillRectangle 画一个小填充方块(简化),或用 DrawText 画 "✓" 字符。用后者最简单。

### 4. Slider 拖拽
OnPointerPressed 记录起始,OnPointerMoved 根据鼠标 X 位置计算 Value = Minimum + (x/width)*(Max-Min)。需要 PointerMoved 事件 —— 当前 Control 有 OnPointerMoved 虚方法,Window 命中测试后调用。

### 5. RadioButton 互斥
点击时,从 Window 逻辑根开始深度遍历,找所有 `RadioButton` 且 `GroupName == this.GroupName` 的,设其 `IsChecked = false`(除自己)。

## 不在 M3 范围
- 文本选择/复制粘贴(需要剪贴板集成,M4)
- TextBox 多行/换行(需要 DrawText layout rect 改造)
- 数据绑定(M4)
- .bpfaml 标记语言(M5)
- 样式的 Trigger/Condition(条件样式,M3.1)
- VSM(可视化状态管理,后续)

## 风险
- TextBox 最复杂(光标 + 编辑键 + 裁剪),容易出 bug。先做基础版(输入 + Backspace),增强后续加。
- 样式系统改 GetValue,影响所有控件。需仔细测试不破坏现有 Button/TextBlock 行为。
