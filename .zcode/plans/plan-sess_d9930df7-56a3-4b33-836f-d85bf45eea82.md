# bpf M2.1 实施计划:布局面板 + Border + 附加属性

## 目标
为 bpf 引入 **Grid / Canvas / DockPanel / Border** 四个布局容器,以及支撑它们所需的 **附加属性(AttachedProperty)** 机制。完成后用户可以写复杂行列/停靠/绝对定位布局,并用 Border 美化。

## 分两阶段实施

### 阶段 A:扩展属性系统 —— 附加属性

附加属性是 WPF/Avalonia 布局的核心(`Grid.SetRow(child, 1)`、`Canvas.SetLeft(child, 100)`)。当前 bpf 只有 `StyledProperty<T>`(属主绑定的属性)。需要新增 **AttachedProperty** —— 值存在**子控件**上,但由**父容器**(Grid/Canvas)读写。

**设计**:`AttachedProperty<TValue>` 继承 `StyledProperty<TValue>`,提供静态 `Register<TOwner>(...)`。值的存储复用 `Control.PropertyValueStore`(已存在),无需新存储结构。`Grid` 用 `child.GetValue(Grid.RowProperty)` 读取。API 与 Avalonia 完全一致。

**新增文件**:
- `src/Bpf/PropertySystem/AttachedProperty.cs` — `AttachedProperty<TValue> : StyledProperty<TValue>`,`static Register<TOwner>(name, defaultValue, affects*)`

**修改**:`StyledProperty.cs` 无需改动(AttachedProperty 是子类)。

### 阶段 B:四个布局面板 + Border

每个面板参照 StackPanel 的模式(`sealed class XPanel : Control, IPanel`,实现 `MeasureCore`/`ArrangeCore`/`Render`/`HitTest`)。

#### 1. Grid(最复杂)
- **字符串 API**:`Grid.Rows = "auto,*,100,2*"`、`Grid.Columns = "auto,*,100"` —— 解析成 `GridLength[]`
- **对象 API**:`Grid.RowDefinitions = new[] { new RowDef(GridLength.Auto), ... }`
- **GridLength**:`struct GridLength { GridUnitType Type; double Value }`,Type ∈ { Pixel, Auto, Star }
- **附加属性**:`Grid.RowProperty`、`Grid.ColumnProperty`、`Grid.RowSpanProperty`、`Grid.ColumnSpanProperty`(用 `child.GetValue(Grid.RowProperty)` 读取)
- **Measure**:两阶段 —— 先量 Auto/Pixel 行高列宽,再把剩余空间按 Star 比例分配
- **Arrange**:按计算后的行高列宽 + RowSpan/ColumnSpan 排列子控件

**新增文件**:
- `src/Bpf/Controls/Grid.cs` — Grid + GridLength + RowDef/ColumnDef
- `src/Bpf/Layout/GridLength.cs` — `struct GridLength`、`enum GridUnitType { Pixel, Auto, Star }`

#### 2. Canvas(绝对定位)
- **附加属性**:`Canvas.LeftProperty`、`Canvas.TopProperty`、`Canvas.ZIndexProperty`
- **Measure**:子控件用 `Infinity` 约束(自由扩展),Canvas 的 DesiredSize = 0(不占用空间)
- **Arrange**:按 `Canvas.GetLeft/Top` 绝对定位,ZIndex 控制绘制顺序
- **Render**:按 ZIndex 排序后绘制

**新增文件**:
- `src/Bpf/Controls/Canvas.cs`

#### 3. DockPanel(停靠)
- **附加属性**:`DockPanel.DockProperty`(Top/Bottom/Left/Right)
- **属性**:`DockPanel.LastChildFill`(bool,默认 true)
- **Measure**:顺序量子控件,主轴按停靠方向累加
- **Arrange**:依次停靠到剩余空间的某一边,LastChildFill=true 时最后一个填满剩余

**新增文件**:
- `src/Bpf/Controls/DockPanel.cs` + `enum Dock { Top, Bottom, Left, Right }`

#### 4. Border(装饰容器,单 Child)
- **属性**:`Background`、`BorderBrush`、`BorderThickness`、`CornerRadius`、`Padding`
- **子控件**:单个 `Child`(不是 IPanel 的 Children 列表)
- **Measure**:量 Child + Padding + BorderThickness
- **Arrange**:Child 排在 BorderThickness+Padding 内缩区域
- **Render**:先画背景(圆角矩形),再画边框,再 push translate 绘制 Child

**新增文件**:
- `src/Bpf/Controls/Border.cs`

### 阶段 C:验证 sample

**修改 `samples/Bpf.Samples.HelloWorld/Program.cs`**:重写为一个演示所有新面板的场景:
- 用 Grid 做两行:顶行放标题 TextBlock,底行放一个 DockPanel
- DockPanel 里 Top 停靠提示文字,Bottom 停靠计数器,中间 Fill 放 Canvas
- Canvas 上绝对定位两个 Button
- 用 Border 包裹计数器(带圆角边框)

## 关键技术点

### 1. 附加属性的存储
AttachedProperty 的值存在**目标控件**(子)上,不是属主(Grid)上。`Grid.SetRow(child, 1)` 等价于 `child.SetValue(Grid.RowProperty, 1)`。`Control.PropertyValueStore` 已支持任意 `BpfProperty` 作 key,所以附加属性零存储改动 —— 只需新的属性类型 + 便捷静态方法。

### 2. GridLength 字符串解析
`"auto,*,100,2*"` → `[Auto, Star(1), Pixel(100), Star(2)]`。解析逻辑:
- `auto`/`auto `(忽略大小写) → Auto
- `*` → Star(1);`2*`/`1.5*` → Star(系数)
- 纯数字 → Pixel(值)

### 3. Grid 的两阶段 Measure
1. 第一遍:Auto 行量子控件取最大高度;Pixel 行直接用声明值
2. 计算剩余空间 = 总高 - Auto - Pixel
3. Star 行按系数瓜分剩余空间

### 4. Canvas.ZIndex 与渲染顺序
Canvas 的 Children 用附加属性 ZIndex 排序后再 Render/HitTest(高 ZIndex 在上层)。HitTest 倒序遍历(上层优先命中)。

### 5. Border 的单 Child 模式
Border 不是 IPanel(无 Children 列表),只有一个 `Child` 属性。它继承 Control,在 `AttachNonPanelChildren` 里 attach 单个 Child(参照 Button 的 TextBlock 模式)。

## 风险
- Grid 的两阶段 Measure + Span 计算逻辑较复杂,容易出 bug。我会先做无 Span 的简单版,Span 作为增强。
- GridLength 字符串解析要处理边界情况(空格、非法值)。
